# Design Editing Workflow

Build a structured, repeatable import-to-delivery workflow that maximizes editing efficiency and maintains file integrity.

## Why This Is Best Practice

**Adopted by:** Adobe's own Lightroom documentation; ASMP Digital Imaging Guide; professional photographers processing 500-5,000 images per session
**Impact:** ASMP Professional Business Practices documents that photographers with defined editing workflows deliver 40% faster and with 60% fewer client revision requests than those using ad hoc post-processing

**Why best:** Without a defined workflow, photographers make inconsistent decisions under pressure: over-editing some images, under-editing others, losing files to poor organization, and delivering inconsistent batches. A workflow converts subjective editing into a repeatable process where decisions about culling, color, and export happen at defined stages, not randomly.

## Steps

1. **Import with structure** — Import to a dated folder (YYYY-MM-DD_ClientName_JobType). Apply import preset: lens correction, camera calibration, copyright metadata, rename files to a consistent pattern (YYYYMMDD_001.RAF). Back up to a second drive on import.
2. **Cull to selects** — Rate images in two passes. Pass 1: flag rejects (X) while reviewing at 100% for focus, blink, and major exposure errors. Pass 2: assign stars (1-3) for quality among keepers. Deliver only 3-star images.
3. **Set global white balance and exposure** — On the first select image: set WB, exposure, highlight/shadow recovery. Sync this base correction to all images from the same lighting setup using "Sync Settings."
4. **Apply a base style or preset** — Apply a starting-point preset that matches the intended look (natural, cinematic, B&W). Presets reduce per-image editing time from 5 minutes to 30 seconds.
5. **Edit selects individually** — Fine-tune each selected image: local adjustments (subject masking, sky masking), specific color corrections, crop and straighten. Never edit rejects.
6. **Apply color grading** — HSL adjustments, tone curve, color grader. Keep the grade consistent across the session unless different looks are required. Reference a delivered set from a prior job to maintain brand consistency.
7. **Run a consistency review** — Sort all selects by capture time and review as a slideshow. Images from the same setup should look identical; outliers require revisiting.
8. **Export for delivery** — Export settings: file format (JPEG 100% for web, TIFF 16-bit for print), color space (sRGB for web, Adobe RGB for print), output sharpening (Standard for web, Low for print). Deliver via the agreed method (gallery link, Dropbox, USB).

## Rules

- Never deliver unculled images — clients cannot select from 600 similar images; editorial judgment is part of the professional service.
- Apply lens correction on import to every image — distortion and vignetting from uncorrected lenses degrade all subsequent edits.
- Edit selects only — editing rejects wastes time; culling discipline is the most valuable efficiency gain in the workflow.
- Export originals as backup before any editing — never destructively edit RAW files; Lightroom non-destructive editing is the standard.

## Examples

Wedding photography workflow (1,200 RAW files): Import with date folder + couple name. Pass 1 cull: flag 400 rejects (blurry, blinking, duplicates). Pass 2: star rate 800 remaining; 3-star selects = 350 images. Sync WB/exposure by lighting setup (ceremony, cocktail, reception = 3 groups). Apply "Wedding Natural" preset. Individual edit time: 20 seconds average. Color grade pass: 30 minutes. Export: 350 JPEGs at sRGB. Total workflow time: 4 hours for 350 polished delivery images.

## Common Mistakes

- Culling and editing simultaneously — slows both processes; separate culling from editing to maintain speed.
- No folder structure — files scattered across the desktop become unrecoverable after 6 months; date+client folder structure is mandatory from day one.
- Editing to 100% zoom for all adjustments — color and tone decisions must be made at fit-to-screen view; 100% zoom is for sharpness and noise evaluation only.
- Skipping the consistency review — delivers batches where image 1 is warm and image 50 is cool; clients notice and request revisions.

## When NOT to Use

- When the assignment is single-image fine-art work where each image is an independent piece with its own creative treatment and batch consistency is neither required nor desired.
- When a client or publisher has an in-house retouching team that receives RAW files and applies their own standardized processing pipeline, making a custom delivery workflow redundant or conflicting.
- When shooting personal rapid-turnaround social content where a minimal edit-and-post approach is intentional and the overhead of a multi-stage structured workflow exceeds the value delivered.
